Fully Reproducible Sales Data Analysis

This project provides summaries and visualizations for January sales data. Using customer lookup information and transaction records, the analysis produces a CSV file with region + category breakdowns for sales, and a bar chart showing total revenue by region. This repository is organized with separater folders for data, scripts, and results to make it easy to navigate and reproduce. :)

Python pandas plotly matplotlib Docker Conda
More details

Problem

Raw sales data is often fragmented across multiple CSV files (transactions, customer lookups, and weather records), making it difficult for stakeholders to quickly assess regional performance. Without a reproducible pipeline, manual data cleaning and visualization lead to inconsistencies and delayed reporting.

Approach

Developed a modular Python pipeline that merges transaction logs with customer regional data. The data is cleaned and aggregated using pandas to calculate metrics like total units, total revenue, and average revenue per order. To ensure portability, the project was containerized using Docker and supports multiple environment setups (venv and Conda). An interactive Plotly visualization was added to allow users to zoom and inspect specific regional data points.

Results & Impact

The pipeline successfully automates the generation of a 12-row summary report and multiple visualizations. By transitioning from static images to interactive Plotly charts, users gained the ability to hover for exact revenue values and zoom into specific regions, significantly improving data accessibility. The Docker integration ensures the analysis runs identically across Windows, Mac, and Linux environments.

NYC 311 Service Request Analysis on AWS

This project analyzes around 200,000 NYC 311 service requests and builds a predictive model to forecast complaint resolution times across agencies and boroughs. Using AWS cloud infrastructure (S3, SageMaker) and machine learning (Linear Learner), the project processes large-scale complaint data, performs exploratory analysis, and develops a regression model. The work demonstrates end-to-end cloud data engineering with stakeholder-focused predictive insights.

Python pandas Jupyter Notebooks AWS (S3, SageMaker, Linear Learner) SQL Plotly scikit-learn Git/GitHub
More details

Problem

NYC 311 complaint data spans millions of requests across multiple agencies and boroughs. City planners need to predict how long it will take to resolve a ticket and identify which complaint types take longest to resolve, enabling better resource allocation.

Approach

Developed an end-to-end ML pipeline using AWS SageMaker's Linear Learner algorithm to predict complaint resolution times. Data ingestion included loading 200k complaints from S3, parsing string dates, handling missing ZIP codes and resolution descriptions, and feature engineering (complaint type, borough, agency, creation date). Exploratory data analysis revealed complaint distributions and temporal patterns. The Linear Learner model was trained/evaluated on engineered features to predict resolution duration, with confusion matrix analysis to assess prediction accuracy across complaint categories.

Results & Impact

Successfully deployed a predictive model that forecasts resolution times for NYC 311 complaints with measurable accuracy. Model insights identify high-volume complaint types and agencies with longer resolution windows. Stakeholder question 3 analysis validates model performance and provides actionable recommendations for prioritizing complaint categories. The cloud-based approach enables retraining on new data and integration with city services.

Sakila DB Revenue Analysis by Country

This project analyzes rental revenue by country using the Sakila rental database, answering the key question: "Which countries generate the highest rental revenue and how should resources be allocated?" The analysis demonstrates advanced SQL techniques including joins, aggregate queries, CTEs, and window functions to provide data-driven insights for resource allocation in a rental business context.

SQL MySQL Docker Docker Compose Jupyter Notebooks Adminer (Database GUI) Git/GitHub
More details

Problem

A rental business using the Sakila database needs to understand which countries generate the highest rental revenue to optimize resource allocation. Without proper analysis, resources might be inefficiently distributed across global markets.

Approach

Set up a local MySQL database using Docker Compose with the Sakila sample database. Performed comprehensive SQL analysis including complex joins to connect rental, payment, customer, and address tables; used aggregate functions with GROUP BY and HAVING clauses to calculate revenue by country; implemented CTEs for modular query structure; applied window functions (RANK, percent of total) for comparative analysis; and utilized EXPLAIN for query optimization. The analysis identifies top revenue-generating countries and provides allocation recommendations.

Results & Impact

Successfully identified countries with highest rental revenue and provided clear recommendations for resource allocation. The analysis demonstrates proficiency in advanced SQL techniques and database query optimization. Stakeholders can now make informed decisions about market focus and resource distribution based on empirical data from the Sakila rental database.
